    User Documentation and Technical Specifications

    Software Details

    This app was built for the purpose of creating an online map that allows the input of a site location and the elevation of a proposed mining cut and creates an updated DEM showing the cut. The app was created in the Tethys Platform using html, CSS, and JavaScript by Christian Kesler and Ashley Hunt.

    Compatibility and Limitations

    This features of this app best function in Google Chrome and Mozilla Firefox. As such, the app should be viewed in either of these browsers. The program does not guarantee to work on every device. Tethys platform runs on Linux and requires many dependencies. For information about how to install Tethys refer to this website: http://docs.tethysplatform.org/en/latest/installation.html. For other issues or troubleshooting assistance, please contact Christian (christiankesler11@gmail.com) or Ashley (lashleyhunt@gmail.com).

    Website Design

    Purpose of Design

    The purpose of this app is create an updated DEM based on a proposed mining cut. We first needed to add a base map to our app so that any features added to the map would have a visual location. Next, we imported all the necessary data files into ArcMap 10.3 and created a working model. We were then able to upload our model to the server and then load it into our Tethys app. After the data was imported, the code was adjusted for formatting. The coding was done in Pycharm and the final map showing an updated DEM is located in the figure below.

    Menu & Interface Elements

    The program is formatted for convenience for users to easily access the map and to navigate through the app. The map can be accessed from the home page through the menu at the left-hand side of the page. The layout of the home page including the menu items can be seen in the figure below.

    The home page informs the user of the purpose of the app. The User Documentation page explains key functions and uses of the app. Lastly, the Help File gives important technical information about the tool.

    Under the Map tab there are three steps. Step 1 allows the user the draw a polygon on the map by clicking on the polygon button.

    In Step 2 the user inputs the desired height at which to cut the land (meters). And in Step 3 the user clicks the ‘Click here to compute volume’ button to run the app.

    Key Functions

    The following table contains all of the functions in the app’s code. The right column indicates the key functions and the left column briefly explains their role in the code.

    Map – allows for the use of a map with adjustable zoom and centering

    SimpleLineSymbol – drew lines that were able to be closed into a polygon

    SimpleFillSymbol – filled the polygon with a color so that it was easy to see

    Geoproscessor – linked to the tool on geoserver.byu.edu

    Button – created a button to click on that runs the app

    esriRequest – used to grab the volume from a text file

    clear – erased the old polygon when the create polygon button was clicked on again
